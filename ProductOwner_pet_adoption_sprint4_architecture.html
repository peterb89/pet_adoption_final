Product Workflow
User Journey from Entry to Adoption
Step-by-step functional flow for all user types. This is the source of truth for what pages and interactions the app must support — no technical details, just product logic.

Registered User Journey

1
Land on HomepageUser arrives at the animal listing page. Browseable without an account.
2
Browse & Filter AnimalsFilter by species, size, gender, age. See animal cards with image, name, species, age.
3
View Animal DetailFull profile: description, requirements, tags, comments. Available to everyone.
4
Register / LoginAccount required to interact. Email + password registration with email verification.
5
Complete ProfileFill in housing type, garden, other pets, experience, motivation. Required before applying.
🔒 Auth required
6
Submit Adoption ApplicationSelect animal, add personal message. One application per animal. Status: pending.
🔒 Auth + Profile required
7
Post CommentsLeave comments on any animal's detail page. Visible to all visitors.
🔒 Auth required
8
Track Application StatusUser can see if their application is pending, approved, or rejected from their profile area.

Admin / Staff Journey

1
Login to Admin PanelAccess via /admin. Secured role. Uses EasyAdmin interface.
🔒 ROLE_ADMIN only
2
Manage AnimalsCreate, edit, delete animals. Set species, breed, size, availability, image, adoption requirements.
3
Manage Species & TagsAdd/edit/remove species types and animal tags used for browsing and filtering.
4
Review Adoption ApplicationsSee all pending applications. Review user profile info. Approve or reject each application.
5
Manage UsersView registered users, check profiles, manage roles.
6
Manage Navbar & Footer⭐ Sprint 4 feature. Edit navigation items from the admin panel. Dynamic layout driven by database.
7
Moderate CommentsReview and remove inappropriate comments from animal pages.
